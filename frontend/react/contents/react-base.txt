


## 1. Estado (State) no React

O **estado (state)** representa dados que podem mudar ao longo do tempo dentro de um componente.

Quando o estado muda, o React **re-renderiza automaticamente** o componente para atualizar a interface.

Características importantes do estado:

- É local ao componente
- Pode mudar durante a execução da aplicação
- Quando atualizado, causa uma nova renderização da UI

Exemplo de uso comum:

- formulários
- listas dinâmicas
- contadores
- filtros
- dados de APIs

---

## 2. useState

`useState` é um **Hook do React** usado para criar e gerenciar estado em componentes funcionais.

Ele retorna dois elementos:

1. O valor atual do estado
2. Uma função para atualizar esse estado


---

## 3. Inputs Controlados

Em React, formulários normalmente utilizam **inputs controlados**.

Isso significa que o valor do input **é controlado pelo estado do React**, não pelo DOM diretamente.

Benefícios:

- controle total dos dados do formulário
- validação mais fácil
- sincronização com outros estados

---

## 4. Eventos no React

Eventos permitem reagir a ações do usuário, como:

- clique
- digitação
- envio de formulário
- hover
- teclado

Alguns exemplos comuns:
onClick
onChange
onSubmit
onKeyDown


Eventos no React utilizam **Synthetic Events**, uma camada de abstração que padroniza o comportamento entre navegadores.

---

## 5. Imutabilidade de Estado

No React, o estado **não deve ser modificado diretamente**.

Em vez disso, deve-se criar **uma nova versão do estado**.

Motivo:

O React detecta mudanças comparando referências de objetos.

Se o mesmo objeto for modificado diretamente, o React pode não perceber a mudança.

Por isso usamos operações como:

- spread operator (`...`)
- map
- filter
- concat


---

## 6. Listas no React

Quando precisamos renderizar vários elementos baseados em dados, utilizamos **arrays e o método map()**.

O `map()` percorre um array e retorna um novo conjunto de elementos JSX.


---

## 7. Key em listas

Cada item renderizado em uma lista precisa de uma **key única**.

A key ajuda o React a identificar quais elementos:

- foram adicionados
- foram removidos
- foram alterados

Sem keys adequadas, o React pode ter problemas de performance ou renderização incorreta.

Boas práticas:

- usar IDs únicos
- evitar usar índice do array quando possível

---

## 8. Componentização

Componentização é o processo de dividir a interface em **pequenos componentes reutilizáveis**.

Benefícios:

- código mais organizado
- maior reutilização
- manutenção mais fácil
- separação de responsabilidades


O componente pai geralmente:

- controla estado
- gerencia lógica

Os componentes filhos geralmente:

- exibem dados
- disparam eventos

---

## 9. Props

Props (properties) são **dados enviados de um componente pai para um componente filho**.

Elas permitem compartilhar informações entre componentes.

Características:

- são somente leitura
- fluem do pai para o filho
- não devem ser modificadas pelo componente filho



---

## 10. Comunicação entre Componentes

Em React, o estado normalmente vive no **componente mais alto da árvore que precisa controlar os dados**.

Esse componente:

- guarda o estado
- envia dados para filhos via props
- envia funções para filhos dispararem eventos

## 11. Manipulação de Arrays no Estado

Quando trabalhamos com listas, usamos métodos de array para atualizar o estado.

### Adicionar item

Criar novo array com os itens antigos + novo item.

### Remover item

Utilizar `filter()` para manter apenas os elementos desejados.

### Atualizar item

Utilizar `map()` para modificar um elemento específico.

Essas operações mantêm a **imutabilidade do estado**, fundamental para o funcionamento correto do React.
